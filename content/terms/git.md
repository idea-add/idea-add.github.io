## 한눈에 보는 정의

**Git**은 중앙 서버 없이도 개발자 각자의 컴퓨터에 전체 변경 이력의 복사본을 갖는 분산형 버전 관리 시스템으로, 오늘날 소프트웨어 개발에서 사실상 표준으로 쓰입니다.

## 자세히 알아보기

- 예문: "개발자들은 각자 로컬 컴퓨터에서 **Git**으로 변경 이력을 관리하다가, 필요할 때 GitHub 같은 원격 저장소에 올려 서로 공유한다."
- {{< term "svn" >}}(SVN)처럼 중앙 서버에 항상 접속해야 하는 것과 달리, 인터넷에 연결되지 않아도 로컬에서 커밋하고 변경 이력을 확인할 수 있어 속도가 빠르고 유연합니다.
- 여러 개발자가 각자의 작업 흐름을 독립적인 브랜치로 나눠 진행하다가, 나중에 이를 다시 합치는 작업이 쉽게 이루어지도록 설계되어 있습니다.
- {{< term "devops" >}}(DevOps)와 {{< term "ci-cd" >}}(CI/CD) 파이프라인 대부분이, 코드가 Git 저장소에 반영(push)되는 것을 시작점으로 삼아 자동으로 빌드와 배포를 트리거하는 방식으로 연동됩니다.
