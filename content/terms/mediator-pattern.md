## 한눈에 보는 정의

**Mediator** 패턴은 여러 객체가 서로를 직접 참조하며 복잡하게 얽히는 대신, 중재자(Mediator) 객체 하나를 거쳐서만 소통하도록 만드는 디자인 패턴입니다.

## 자세히 알아보기

- 예문: "관제탑을 통해서만 소통하는 비행기들처럼, 여러 UI 요소가 서로 직접 참조하지 않고 **Mediator** 하나를 통해서만 상호작용하게 만든다."
- 객체가 늘어날수록 서로를 직접 참조하는 관계의 수가 기하급수적으로 늘어나는데, 중재자를 두면 모든 객체가 중재자하고만 관계를 맺으면 되어 구조가 훨씬 단순해집니다.
- 각 객체는 중재자에게 알리기만 하면 되고, 그 알림을 받아 다른 객체에 전달할지 판단하는 로직은 중재자가 전담합니다.
- {{< term "external-coupling" >}}(결합도)의 관점에서 보면, 객체 사이의 결합을 낮추고 중재자에게 로직을 집중시키는 설계 전략이라고 할 수 있습니다.
