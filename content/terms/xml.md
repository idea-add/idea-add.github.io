## 한눈에 보는 정의

**XML**(Extensible Markup Language)은 `<태그>`를 이용해 데이터의 구조와 의미를 함께 표현하는 마크업 언어로, 서로 다른 시스템끼리 데이터를 주고받을 때 공통 형식으로 널리 활용되었습니다.

## 자세히 알아보기

- 예문: "`<name>홍길동</name>`처럼 **XML**은 태그로 데이터가 무엇을 의미하는지 함께 표현한다."
- 태그 이름을 자유롭게 정의할 수 있어(Extensible), 업종이나 시스템마다 필요한 데이터 구조를 유연하게 표현할 수 있었습니다.
- 서로 다른 회사의 시스템을 연결하는 {{< term "eai" >}}(전사적 애플리케이션 통합) 환경에서 데이터 교환 형식으로 오랫동안 널리 쓰였습니다.
- 최근에는 문법이 더 간결하고 가벼운 JSON 형식이 웹 {{< term "api" >}} 통신에서 XML을 상당 부분 대체했지만, 문서 형식이나 일부 전통적인 기업 시스템에서는 여전히 활용되고 있습니다.
