## 한눈에 보는 정의

**논리적 데이터 모델**(Logical Data Modeling)은 {{< term "conceptual-data-model" >}}(개념적 데이터 모델)을 바탕으로, 특정 {{< term "dbms" >}}(DBMS) 제품에 얽매이지 않으면서도 각 개체가 가질 속성과 개체 간 관계를 구체적으로 정의하는 단계입니다.

## 자세히 알아보기

- 예문: "'회원' 개체에 이름, 이메일, 가입일 같은 속성을 정의하고, {{< term "pk" >}}(기본키)와 {{< term "fk" >}}(외래키)를 지정하는 것이 **논리적 데이터 모델** 단계다."
- {{< term "erd" >}}(ERD)로 이 단계의 결과물을 시각적으로 표현하는 경우가 많으며, 정규화를 통해 데이터 중복을 줄이는 작업도 이 단계에서 이루어집니다.
- 아직 어떤 DBMS 제품을 쓸지는 확정하지 않은 상태로, 이론적으로 올바른 데이터 구조를 설계하는 데 집중합니다.
- 이 논리 모델을 실제 사용할 DBMS의 문법과 기능에 맞게 구체화하면, 다음 단계인 {{< term "physical-data-model" >}}(물리적 데이터 모델)이 됩니다.
