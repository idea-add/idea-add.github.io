## 한눈에 보는 정의

**정적 테스트**(Static Testing)는 프로그램을 실제로 실행하지 않고, 코드 리뷰나 정적 분석 도구를 통해 소스 코드와 문서를 검토하며 오류를 찾아내는 테스트 방식입니다.

## 자세히 알아보기

- 예문: "코드를 배포하기 전, 동료들이 함께 코드를 검토하는 코드 리뷰도 **정적 테스트**의 한 형태다."
- {{< term "srs" >}}(요구사항 명세서) 같은 문서가 앞뒤가 맞는지 검토하는 것부터, 코드 스타일이나 잠재적 버그 패턴을 자동으로 찾아주는 정적 분석 도구 활용까지 다양한 형태로 이루어집니다.
- 실제로 프로그램을 돌려 오류를 확인하는 {{< term "dynamic-testing" >}}(동적 테스트)보다 앞선 단계에서, 더 적은 비용으로 문제를 조기에 발견할 수 있다는 장점이 있습니다.
- {{< term "white-box-testing" >}}과 마찬가지로 코드 내부를 직접 들여다본다는 공통점이 있지만, 정적 테스트는 코드를 실행하지 않는다는 점에서 구분됩니다.
