## 한눈에 보는 정의

**정보공학 방법론**(Information Engineering Development)은 개별 시스템 하나가 아니라 조직 전체의 데이터를 먼저 분석하고 표준화해, 여러 시스템이 일관된 데이터 구조를 공유하도록 설계하는 개발 방법론입니다.

## 자세히 알아보기

- 예문: "이 은행은 여러 부서의 시스템을 각각 따로 만들지 않고, **정보공학 방법론**에 따라 전사 데이터 모델을 먼저 정의한 뒤 시스템을 구축했다."
- {{< term "erd" >}}(ERD) 같은 데이터 모델링 도구를 활용해 조직 전체의 데이터를 먼저 표준화하고, 이를 바탕으로 여러 응용 시스템을 일관되게 개발하는 데이터 중심 접근 방식입니다.
- 기능(절차) 중심으로 설계하는 {{< term "structured-development" >}}(구조적 방법론)과 달리, 데이터 구조를 먼저 정의하고 그 위에 기능을 얹는다는 점에서 접근 순서가 다릅니다.
- 여러 시스템이 흩어져 있는 대규모 조직에서, 데이터 정합성과 재사용성을 높이기 위한 목적으로 주로 채택됩니다.
