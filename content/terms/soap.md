## 한눈에 보는 정의

**SOAP**(Simple Object Access Protocol)은 {{< term "xml" >}}(XML) 형식으로 메시지의 구조를 엄격하게 정의해, 서로 다른 시스템 간에 데이터를 주고받는 웹 서비스 통신 프로토콜입니다.

## 자세히 알아보기

- 예문: "이 금융 시스템은 보안과 신뢰성이 중요해, 지금도 일부 연동 구간에서 **SOAP** 방식을 사용한다."
- 메시지 형식, 오류 처리, 보안 규칙까지 매우 엄격한 표준으로 정해져 있어, 안정성과 신뢰성이 중요한 기업 시스템 연동에 적합합니다.
- {{< term "rest" >}}(REST)보다 형식이 훨씬 복잡하고 무거워, 오늘날 일반적인 웹·모바일 API 개발에서는 REST나 {{< term "json" >}}(JSON) 기반 방식이 훨씬 널리 쓰입니다.
- 은행, 공공기관처럼 엄격한 표준과 검증 절차가 요구되는 시스템에서는 지금도 SOAP 방식의 연동이 유지되는 경우가 있습니다.
