## 한눈에 보는 정의

**API Gateway**는 외부(클라이언트, 사용자)에서 들어오는 API 요청을 받아, 인증·요청 제한(rate limiting) 등을 처리한 뒤 알맞은 내부 서비스로 전달해주는 '관문' 역할의 시스템입니다.

## 자세히 알아보기

- 클라이언트 입장에서는 여러 마이크로서비스가 있어도 API Gateway 주소 하나만 알면 됩니다.
- 인증({{< term "jwt-oauth2" >}} 처리 포함), 로깅, 요청 속도 제한 같은 공통 기능을 각 서비스마다 반복 구현하지 않고 게이트웨이에서 한 번에 처리할 수 있습니다.

### Q. Service Mesh와는 어떻게 다른가요?

**API Gateway**는 '외부 → 내부'로 들어오는 요청(남북 트래픽)을 관리하는 정문이고, {{< term "service-mesh" >}}는 '내부 서비스끼리' 주고받는 요청(동서 트래픽)을 관리하는 내부 통신망입니다. 두 가지는 함께 쓰이는 경우가 많습니다.
